FreeXplorer 1.1

Interface de commande pour Freebox et VLC permettant de :

  • jouer des fichiers (musiques, images, videos) stockées sur le PC
  • naviguer dans les disques et les repertoires du PC
  • visionner des DVD insérés dans le lecteur de DVD du PC
  • avancer et reculer facilement dans les videos comme dans les DVD
  • rejouer facilement un des derniers fichiers recemment joués
  • naviguer sur Internet (sur des sites "light" adaptés à la Freebox ou au WAP/iMode)
  • ecouter des WebRadios ou visionner des WebTV
  • conserver ses favoris et jouer des playlist
  • piloter d'autres applications PC avec la télécommande

Explication sur le format des fichiers et les fonctions

(pour ceux qui souhaiterait retoucher les pages html standard) 
Le serveur HTTP est très basique et répondra à toutes les requetes pour des fichiers .gif .htm .html .xml
en allant chercher le fichier dans le repertoire de l'executable.
Il n'accepte des requetes qu'en provenance de la Freebox elle-meme (pas de risque d'etre utilisé par qqun depuis Internet)
Pour le format HTML de la freebox, se référer au document .PDF fourni avec le Freeplayer
Si un fichier HTML est accompagné d'un fichier du meme nom mais avec l'extension .arg, ce dernier contient la liste
des paramètres à remplacer dans le fichier HTML (ex: la 3eme ligne du .arg identifie ce que contiendra le tag {3} dans l'HTML)
Chaque ligne du .ARG peut avoir 3 formes :
- un identifiant => le tag sera remplacé par la valeur de l'identifiant passé dans l'URL
	ex: param		=> /mapage.html?param=coucou		=> le tag {x} sera remplacé par coucou
- une variable globale => le tag sera remplacé par la valeur de la variable globale
	le nom des variables commencent par un underscore '_'
	pour changer la valeur d'une variable globale, il suffit que son nom apparaisse en parametre d'une URL
	sa valeur est ensuite conservée meme si les URL ulterieures n'y font plus reference
		ex: /mapage.html?_file=toto
	ex: _file		=> le tag {x} sera remplacé par la derniere valeur de _file donnée dans une URL
- une fonction (eventuellement suivi d'un espace et d'un paramètre qui peut lui meme etre une fonction a son tour)
	ex: $dir param		=> /mapage.html?param=C:\toto.txt	=> le tag {x} sera remplacé par C:\

les fonctions disponibles sont :
"$explore": // renvoit l'URL a utiliser suivant le contenu de la variable globale _file
"$folders"+param: // HTML représentant la liste des dossiers avec comme dossier actif celui désigné
"$files"+param: // HTML représentant la liste des fichiers du repertoire contenant le(s) fichier(s) désigné(s)
"$fullurl": // retourne URL+Query (à l'exception de /settings.html qui devient /home.html)
"$dir" + param: // repertoire contenant le fichier désigné par la variable globale _file
"$url" + param: // version du parametre pour être placé dans une URL
"$audio_mode": // mode audio dans lequel placer la Freebox
